128. Longest Consecutive Sequence#

After watching this, I literally converted the solution to Go. I think it is not the best, but it is the first thing came to my mind. Here is the code:



// NOTE: Read the explanation below first ‚¨áÔ∏è

// longestConsecutive returns the length of 
// the longest consecutive elements sequence.
func longestConsecutive(nums []int) int {
    // we create a set of nums
    set := make(map[int]struct{})
    // we put all values of nums into the set
    for _, num := range nums {
        set[num] = struct{}{}
    }
    // longest is used to store the longest sequence  which 
    // is going to be returned at the end of the function
    longest := 0

    // we look at each element in the nums array
    for _, num := range nums {
        // we check if the current number is the start of the sequence;
        // in order to be a start of the sequence, there should be no
        // num-1 in the set
        if _, ok := set[num-1]; !ok {
            // once we are sure that `num` is the start of the sequence,
            // we try to find the length of this sequence, which is initially 0
            length := 0
            // we check if there is consecutive numbers in the set by 
            // incrementing the length, if there is a consecutive number
            // `isConsecutive` is true, that means we should keep looking
            // for the next consecutive numbers
            isConsecutive := true
            for isConsecutive {
                // we check if there is num+length in the set
                if _, ok := set[num+length]; ok {
                    length++
                // if there is no num+length in the set, that means
                // we reached the end of the sequence, so we stop     
                } else {
                    isConsecutive = false
                }
            }
            // we update the longest sequence
            longest = max(longest, length)
        }
    }
   // we are done looking at each element in the nums array
   // now it is time to return the longest sequence
   return longest
}


// no need to explain this function :)
func max(a,b int) int {
   if a > b {
      return a
   }
   return b
}

Explanation: We look at each element num in the array and see if there is num-1 in the nums array. This checking requires a set that has all the elements of nums array. Set in golang can be implemented as a map with empty struct as a value. The rest of the explanantion is in the code comments.

554. Brick Wall#

After watching half of this, I came up with the following solution:

func leastBricks(wall [][]int) int {
    // create a map to store the number of gaps at each position of x line.
    m := make(map[int]int,0)
    // iterate over each layer of the wall
    for _, layer := range wall {
        // pos is the position of the gap
        pos := 0
        // brickNums is the number of bricks in the current layer
        brickNums := len(layer)
        // we iterate over each brick in the current layer 
        for i, brick := range layer {
            // we check if we are not at the end of the layer
            if i != brickNums-1 {
                // if we are not at the end of the layer, we update the position 
                pos = pos + brick
                // we increment the number of gaps at the current position
                m[pos]++
            }
        }
    }
    height := len(wall)
    // we find the maximum number of gaps at any position
    max := 0
    // we iterate over the map and find the maximum number of gaps
    for _, v := range m {
        // if the current number of gaps is greater than the max, we update the max
        if v > max {
            max = v
        }
    }
    // the answer is the height of the wall minus the maximum number of gaps
    return height-h
}

Explanation: The idea is to count the number of gaps on each y line crossing at each units of x line. We iterate over each layer of the wall and store the number of gaps in a map. The key of the map is the position of the gap and the value is the number of gaps at that position. The answer is the height of the wall minus the maximum number of gaps at any position.

560. Subarray Sum Equals K#

I had a hard time understanding the solution of this problem:


func subarraySum(nums []int, k int) int {
   cumulativeSumFreq := make(map[int]int, 0)
   cumulativeSumFreq[0] = 1
   var (
       subArrayCount        int
       currentCumulativeSum int 
   )
   for _, num := range nums {
       currentCumulativeSum += num
       if val, ok := cumulativeSumFreq[currentCumulativeSum-k]; ok {
           subArrayCount += val
       }
       cumulativeSumFreq[currentCumulativeSum]++
   }
   return subArrayCount                              
}

Explanation: The idea is to use a map to store the frequency of cumulative sums. We iterate over each element of the nums array and update the cumulative sum. We use the following mathematical equation as the main idea of the solution:

   x+k = y

here y is the current cumulative sum and it is dynamic variable. Our goal is to find the number of x whose sum with k is equal to y which is a running sum.
